title: "Error Codes"
description: "Standard HTTP error codes you may encounter in API responses."
---

Holiday API uses standard HTTP status codes to indicate the result of operations.

## Status Codes

| Code | Description |
| :--- | :--- |
| `200` | **Success:** Request was processed successfully. |
| `400` | **Bad Request:** Parameters are missing or in invalid format (e.g., invalid date format, missing country code). |
| `401` | **Unauthorized:** API key is invalid, missing, or quota exceeded. |
| `403` | **Forbidden:** You don't have permission to make this request. |
| `404` | **Not Found:** The requested resource (country, year, etc.) does not exist. |
| `429` | **Too Many Requests:** Your per-minute or daily request limit has been exceeded. Please try again later. |
| `500` | **Server Error:** An unexpected error occurred on the server. |
| `503` | **Service Unavailable:** External data providers are unreachable or maintenance is in progress. |

## Error Response Format

API errors typically return a JSON object conforming to the [Problem Details (RFC 7807)](https://tools.ietf.org/html/rfc7807) standard.

```json
{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "Bad Request",
  "status": 400,
  "detail": "Invalid country code: 'XX'",
  "instance": "/api/calendar/XX/2025"
}
```
